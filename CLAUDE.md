## Running the App

```bash
python -m venv .venv
source .venv/bin/activate        # Linux/macOS
.venv\Scripts\Activate.ps1       # Windows PowerShell
pip install -r requirements.txt
python main.py
```

External runtime dependencies (must be on PATH): **FFmpeg** (required), Resolume codec (optional, for DXV encoding).

There is no formal test suite or lint configuration.

## Architecture Overview

TELE-TAG is a PyQt6 desktop app for managing local video libraries with hierarchical tagging and FFmpeg-based re-encoding.

### Layer responsibilities

| Layer | Path | Role |
|---|---|---|
| Entry point | `main.py` | Qt bootstrap, disables GPU decode for stability, applies saved theme |
| DB | `teletag/db/` | Thread-local WAL-mode SQLite connections (`connection.py`) + DDL/migrations (`schema.py`) |
| Core | `teletag/core/` | Business logic, no Qt imports |
| UI | `teletag/ui/` | PyQt6 widgets; communicates with core via signals/slots |

### Core modules

- **`library.py`** — Library CRUD; each library is a folder with a `.teletag/metadata.db` inside.
- **`ingest.py`** — Scan → xxhash64 → ffprobe metadata → thumbnail extraction → DB upsert.
- **`tags.py`** — Hierarchical tags implemented with a **closure table** (`tag_closure`: ancestor_id, descendant_id, depth). Use this for O(1) descendant lookups; always update the closure table when adding/removing/reparenting tags.
- **`encode.py`** — FFmpeg subprocess runner; presets (HAP, DXV, H.264, ProRes, WebM) + scale modes; progress parsed from FFmpeg stdout.
- **`watcher.py`** — watchdog-based FS monitor; emits Qt signals (`file_added`, `file_deleted`, `file_moved`).

### DB schema (key tables)

- `files` — video metadata keyed by `xxhash` + `relative_path`
- `tags` + `tag_closure` — hierarchical tag tree
- `file_tags` — many-to-many file ↔ tag
- `encode_jobs` — re-encode queue with status

### Threading model

- **Main thread**: Qt event loop only
- **Worker threads** (QThread): ingest scanning, encode jobs
- **Daemon thread**: watchdog file watcher
- All DB access uses thread-local connections from `connection.py` — never share a connection across threads.

### UI layout

`MainWindow` has two tabs: **Library** (tag_panel left | grid_panel center | detail_panel right) and **Convert** (convert_panel). Themes are QSS stylesheets generated by `theme.py`; four built-in themes (Aura, VS Dark, Dracula, Light) stored via QSettings.
